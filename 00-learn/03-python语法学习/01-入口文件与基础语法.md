# 01 入口文件与基础 Python 语法

> 对应代码：`nanobot/__init__.py`、`nanobot/__main__.py`、`nanobot/cli/commands.py`（前 31 行）

---

## 一、三引号字符串（模块文档字符串）

```python
"""
nanobot - A lightweight AI agent framework
"""
```

- 用 `"""` 或 `'''` 包裹的字符串，可以跨多行
- 放在文件/函数/类的**第一行**，叫做 **docstring（文档字符串）**
- 不会被执行，主要给人和工具（如 `help()`）读的

---

## 二、双下划线变量（Dunder 变量）

```python
__version__ = "0.1.4.post3"
__logo__ = "🐈"
```

- `__xxx__` 格式是 Python 的**约定俗成**，表示"特殊变量"
- 常见的：

| 变量 | 含义 |
|------|------|
| `__version__` | 包的版本号 |
| `__name__` | 模块当前的名字（见下一节）|
| `__file__` | 当前文件的路径 |
| `__init__` | 类的初始化方法 |

---

## 三、`if __name__ == "__main__"`

```python
# __main__.py
if __name__ == "__main__":
    app()
```

- `__name__` 是 Python 内置变量
- **直接运行**该文件时：`__name__ == "__main__"` → **True**，会执行 `app()`
- **被 import 时**：`__name__ == "nanobot"` → **False**，不执行

**实际效果：**
```bash
python -m nanobot     # __name__ = "__main__" → 执行 app()
import nanobot        # __name__ = "nanobot"  → 不执行 app()
```

---

## 四、import 的两种写法

```python
import asyncio              # 导入整个模块，用时写 asyncio.xxx
from pathlib import Path    # 从模块中只拿某一个类/函数，直接用 Path
from nanobot import __logo__, __version__  # 可以同时导入多个
```

**区别：**
- `import X` → 用时需要写 `X.something`
- `from X import Y` → 直接用 `Y`，更简洁

---

## 五、f-string（格式化字符串）

```python
help=f"{__logo__} nanobot - Personal AI Assistant"
# 输出：🐈 nanobot - Personal AI Assistant
```

- 字符串前加 `f`，`{}` 里写变量或表达式，运行时自动替换
- 比旧写法 `"%s" % var` 或 `"{}".format(var)` 更简洁直观

---

## 六、集合（set）

```python
EXIT_COMMANDS = {"exit", "quit", "/exit", "/quit", ":q"}
```

- 用 `{}` 包起来，元素之间用逗号隔开，**没有键值对**
- 特点：**无序、不重复、查找极快（O(1)）**
- 与 `list` 的对比：

| | list `[]` | set `{}` |
|---|---|---|
| 有序 | ✅ | ❌ |
| 可重复 | ✅ | ❌ |
| 查找速度 | 慢（O(n)）| 快（O(1)）|
| 适合场景 | 需要顺序 | 判断"是否存在" |

- 用法：
```python
"exit" in EXIT_COMMANDS   # True，判断是否是退出命令
"hello" in EXIT_COMMANDS  # False
```

---

## 七、类型注解（Type Hints）

```python
_PROMPT_SESSION: PromptSession | None = None
_SAVED_TERM_ATTRS = None
```

- `变量名: 类型 = 值` 是 Python 3.6+ 的**类型注解**语法
- `PromptSession | None` 表示这个变量可以是 `PromptSession` 对象，也可以是 `None`（Python 3.10+ 写法）
- 类型注解**不会强制检查**，只是给人和工具看的提示

## 八、`typer.Typer()` 创建 CLI 应用

**Typer 就是专门用来做命令行工具（CLI）的库**，让你用写普通 Python 函数的方式，自动生成带参数解析、帮助文档的命令行程序。

### 1. 创建根命令对象

```python
app = typer.Typer(
    name="nanobot",
    help=f"{__logo__} nanobot - Personal AI Assistant",
    no_args_is_help=True,   # 啥命令都不加时自动显示帮助
)
```

### 2. 注册子命令（装饰器语法）

```python
@app.command()          # 打上这个装饰器，函数名就是命令名
def agent(
    message: str = typer.Option(None, "--message", "-m"),
):
    ...
```

用户在终端输入 `nanobot agent -m "你好"`，Typer 会**自动**：
1. 解析 `-m` 参数，把 `"你好"` 赋值给 `message` 变量
2. 参数类型写错时自动报错提示
3. 没有参数时显示帮助（因为设置了 `no_args_is_help=True`）

你不用自己写 `sys.argv` 那些繁琐的解析代码，Typer 全帮你做了。

### 3. nanobot 命令总览

每一个加了 `@app.command()` 的函数就是一条命令：

| 函数名 | 对应终端命令 |
|--------|------------|
| `def onboard()` | `nanobot onboard` |
| `def gateway()` | `nanobot gateway` |
| `def agent()` | `nanobot agent` |
| `def status()` | `nanobot status` |

## 总结：程序启动链路

```
用户输入：nanobot agent -m "你好"
          ↓
pyproject.toml [scripts] → nanobot/cli/commands.py 的 app()
          ↓（等效）
python -m nanobot → __main__.py → app()
          ↓
typer 解析命令行参数，调用对应的函数（如 agent()）
```
