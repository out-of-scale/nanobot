## 一、目录结构一览

```
nanobot/
├── __init__.py          ← 版本号、logo 等全局常量
├── __main__.py          ← python -m nanobot 的入口
│
├── cli/                 ← 命令行入口
│   └── commands.py      ← 所有 CLI 命令（agent / onboard / status …）
│
├── config/              ← 配置系统
│   ├── schema.py        ← 配置文件的数据结构定义
│   └── loader.py        ← 从磁盘读取 YAML 配置文件
│
├── bus/                 ← 消息总线（模块间通信的"快递系统"）
│   ├── events.py        ← 消息的数据结构（InboundMessage / OutboundMessage）
│   └── queue.py         ← MessageBus：异步队列的收发逻辑
│
├── agent/               ← 核心 AI Agent 大脑
│   ├── loop.py          ← AgentLoop：主循环，驱动整个 AI 对话
│   ├── context.py       ← ContextBuilder：拼装发给 LLM 的上下文
│   ├── memory.py        ← MemoryStore：对话历史的存储与读取
│   ├── skills.py        ← SkillsLoader：加载/管理外部技能插件
│   └── tools/           ← 内置工具（bash、文件读写等），供 Agent 调用
│
├── providers/           ← LLM 接口封装（统一对接各种 AI 模型）
│   ├── base.py          ← LLMProvider 抽象基类（定义通用接口）
│   ├── litellm_provider.py     ← 通用接口，支持 OpenAI/Claude/Gemini 等百种模型
│   ├── openai_codex_provider.py← 专门对接 OpenAI Codex（旧版）
│   ├── custom_provider.py      ← 用户自定义 provider
│   ├── registry.py      ← provider 注册表，根据配置选择用哪个
│   └── transcription.py ← 语音转文字（Whisper 等）
│
├── channels/            ← 各平台接入层（消息"快递员"）
│   ├── base.py          ← BaseChannel 抽象基类
│   ├── manager.py       ← ChannelManager：管理所有启用的 channel
│   ├── telegram.py      ← Telegram Bot 接入
│   ├── discord.py       ← Discord Bot 接入
│   ├── slack.py         ← Slack Bot 接入
│   ├── feishu.py        ← 飞书接入
│   ├── dingtalk.py      ← 钉钉接入
│   ├── email.py         ← 邮件接入
│   ├── matrix.py        ← Matrix 协议接入
│   ├── mochat.py        ← 企业微信接入
│   ├── whatsapp.py      ← WhatsApp 接入
│   └── qq.py            ← QQ 接入
│
├── session/             ← 会话管理（记住"你是谁"）
│   └── manager.py       ← Session / SessionManager：每个用户一个 session
│
├── skills/              ← 外部技能插件目录（可扩展）
│   ├── weather/         ← 查天气技能
│   ├── github/          ← GitHub 操作技能
│   ├── memory/          ← 长期记忆技能
│   ├── summarize/       ← 内容摘要技能
│   ├── cron/            ← 定时任务技能
│   └── ...              ← 更多技能
│
├── cron/                ← 定时任务系统
│   ├── service.py       ← CronService：定时触发 Agent 执行任务
│   └── types.py         ← CronJob / CronSchedule 数据结构
│
├── heartbeat/           ← 心跳服务（定期唤醒 Agent）
│   └── service.py       ← HeartbeatService：周期性发送唤醒信号
│
├── utils/               ← 通用工具函数
│   └── helpers.py       ← 零散的辅助函数
│
└── templates/           ← 提示词模板（system prompt 等）
```

## 二、各子模块职责表

| 模块 | 核心类/文件 | 一句话职责 | 类比 |
|------|------------|-----------|------|
| `cli/` | `commands.py` | 解析用户在终端输入的命令，是程序的"大门" | 前台接待 |
| `config/` | `schema.py`, `loader.py` | 读取并验证 `config.yaml`，给全程序提供配置数据 | 公司规章手册 |
| `bus/` | `MessageBus` | 模块间传递消息的异步队列，把收消息和处理消息解耦 | 公司内部快递 |
| `agent/` | `AgentLoop` | AI 的大脑：接收消息→调用 LLM→决定用哪些工具→输出 | 核心决策者 |
| `providers/` | `LLMProvider` | 封装调用各种 AI 模型的接口，屏蔽不同 API 的差异 | AI 模型适配器 |
| `channels/` | `ChannelManager` | 接入各平台（Telegram/微信等），把平台消息格式转为内部格式 | 各语言翻译员 |
| `session/` | `SessionManager` | 为每个用户创建和维护对话上下文（记住是谁在说话） | 客服档案系统 |
| `skills/` | 各子目录 | 可插拔的外部技能包（天气/GitHub 等），供 Agent 按需加载 | 技能插件商店 |
| `cron/` | `CronService` | 定时触发 Agent 执行预设任务（如每天早报） | 闹钟/日程表 |
| `heartbeat/` | `HeartbeatService` | 周期性发送心跳，唤醒 Agent 做周期性检查 | 定时巡逻 |
| `utils/` | `helpers.py` | 零散通用工具函数，被其他模块复用 | 工具箱 |
| `templates/` | 各 `.txt` 文件 | 存放 system prompt 等提示词模板 | 话术模板库 |

## 三、消息流转图（核心数据流）

```
用户发消息（Telegram/微信/终端…）
        ↓
  channels/（各平台 Channel）
  把平台消息 → 转为 InboundMessage
        ↓
  bus/（MessageBus 消息队列）
  InboundMessage 放入队列
        ↓
  agent/loop.py（AgentLoop）
  从队列取消息 → 交给 ContextBuilder 拼上下文
        ↓
  providers/（LLMProvider）
  调用 AI 模型（GPT/Claude/Gemini…）获得回复
        ↓
  agent/loop.py（AgentLoop）
  解析回复 → 判断是否需要调用工具
        ↓（如需工具）
  agent/tools/ 或 skills/（内置/外部工具）
  执行工具，结果再喂回 LLM，循环直到得出最终答案
        ↓
  bus/（MessageBus）
  把 OutboundMessage 放入队列
        ↓
  channels/（各平台 Channel）
  把回复发回给用户
```

## 四、模块依赖关系（谁依赖谁）

```
cli/commands.py          ← 最顶层，启动所有服务
    ├── config/          ← 最先读取，提供全局配置
    ├── bus/             ← 消息的传输通道
    ├── channels/        ← 外部平台接入，依赖 bus
    ├── agent/           ← 核心逻辑，依赖 bus + providers + session
    │   ├── providers/   ← 只负责调用 LLM，相对独立
    │   ├── session/     ← 提供用户身份识别
    │   └── skills/      ← 按需加载的技能插件
    ├── cron/            ← 定时任务，依赖 agent
    └── heartbeat/       ← 心跳服务，依赖 bus
```

## 五、学习路线（建议顺序）

| 顺序 | 文件 | 为什么先学它 |
|------|------|------------|
| ① | `cli/commands.py` | 程序入口，能看到所有模块如何被组织、启动 |
| ② | `config/schema.py` | 配置结构是整个项目的"数据蓝图"，先懂配置，后续模块都能对号入座 |
| ③ | `bus/queue.py` | 消息总线是模块间通信的核心，理解它才能懂数据怎么流动 |
| ④ | `agent/loop.py` | 大脑核心，最复杂也最关键；有了前三步知识可以读懂 |
| ⑤ | `providers/` | LLM 接口层，相对独立，学完可立刻看懂 AI 调用方式 |
| ⑥ | `channels/` | 平台适配层，结构重复度高，学一个就能看懂其他 |
| ⑦ | `skills/` + `tools/` | 掌握扩展机制，能自己写插件 |
